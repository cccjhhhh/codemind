# 修改压缩逻辑检查清单

## 每次修改压缩逻辑前，逐一确认以下约束：

### 1. 路径唯一性
- [ ] 是否经过 ContextCompressionOrchestrator 唯一入口？
- [ ] 是否还有其他路径在做同样的事？

### 2. 消息结构
- [ ] Read 工具结果索引是否在管线开始时一次扫描？
- [ ] 所有阶段是否复用同一组 protected indices？
- [ ] L4 摘要后是否保留了消息 role 结构？

### 3. 数据丢失
- [ ] 被压缩的消息内容是否有恢复方式？
- [ ] ASSISTANT-TOOL 配对关系是否完整？

### 4. 时序
- [ ] L1-L2-L3-(L4) 的执行顺序是否正确？
- [ ] L4 摘要后是否跳过后面的 L1-L3？
